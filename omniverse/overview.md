## Overview

This preserved overview page supports existing links to the Omniverse learning sample while positioning the work toward OpenUSD and digital twin content workflows.

The project explores how an Omniverse Kit-based app is created from NVIDIA's Kit App Template and how developer-facing guidance can make the workflow repeatable for technical users. I am using this learning path to build role readiness around OpenUSD asset structure, composition, metadata, packaging, validation, publishing, and reuse.

## What the sample shows

- Getting started flow for creating an Omniverse Kit-based app
- Developer workflow guidance for setup, project structure, and extensions
- Learning-oriented documentation for OpenUSD/Omniverse concepts
- A bridge between developer documentation systems and reusable digital twin asset workflows
- A foundation for explaining packaging, validation, publishing, and discovery patterns for OpenUSD assets

## Why it matters for digital twins

Digital twin asset workflows need more than instructions. They need structured intake, metadata requirements, readiness checks, publishing standards, and discoverability patterns. This learning project supports that transition by connecting Omniverse/OpenUSD concepts to content workflow architecture.
